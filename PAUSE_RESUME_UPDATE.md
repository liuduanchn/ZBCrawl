# 招标信息爬取功能更新说明

## 更新时间
2026-01-22

## 更新内容

### 1. 新增手动暂停功能
- 支持在爬取过程中手动暂停
- 点击"暂停爬取"按钮触发
- 完成当前学校后停止
- 自动保存进度

### 2. 新增自动暂停功能
- 单次爬取时长超过 15 分钟自动暂停
- 防止长时间占用服务器资源
- 保存当前进度，可随时恢复

### 3. 新增恢复爬取功能
- 点击"恢复爬取"按钮从暂停位置继续
- 无需从头开始
- 计时器重置

## 功能改进

### 前端界面
1. **新增暂停按钮**
   - 在爬取过程中显示
   - 点击后发送暂停指令
   - 显示"暂停指令已发送"提示

2. **新增恢复按钮**
   - 在暂停状态下显示
   - 点击后恢复爬取
   - 重置计时器

3. **优化按钮状态**
   - 根据爬取状态动态显示按钮
   - 爬取中：暂停 + 停止按钮
   - 暂停中：恢复按钮
   - 未爬取：开始按钮

4. **增强暂停提示**
   - 显示暂停状态标识
   - 显示暂停原因
   - 显示已完成统计数据

### 后端API
1. **新增暂停API**
   - 路径：`POST /api/tenders/crawl/pause`
   - 功能：发送暂停指令
   - 状态：更新数据库状态为 paused

2. **新增获取暂停状态API**
   - 路径：`GET /api/tenders/crawl/pause`
   - 功能：查询当前暂停状态
   - 返回：paused 状态和 status

3. **新增恢复API**
   - 路径：`DELETE /api/tenders/crawl/pause`
   - 功能：恢复爬取
   - 状态：重置数据库状态为 running

### 爬取逻辑
1. **暂停检测**
   - 每处理一个学校前检查暂停状态
   - 从数据库读取当前状态
   - 发现暂停立即停止

2. **超时检测**
   - 记录爬取开始时间
   - 每处理一个学校前检查超时
   - 超过 15 分钟自动暂停

3. **超时配置**
   - 当前配置：15 分钟
   - 可修改 `MAX_CRAWL_DURATION` 常量
   - 建议范围：10-30 分钟

## 使用方法

### 手动暂停
1. 点击"开始爬取"
2. 等待几所学校爬取完成
3. 点击"暂停爬取"
4. 等待当前学校爬取完成
5. 显示"爬取已暂停"
6. 点击"恢复爬取"继续

### 自动暂停
1. 点击"开始爬取"
2. 等待超过 15 分钟
3. 自动显示"爬取已自动暂停"
4. 点击"恢复爬取"继续

### 查看状态
1. 暂停时显示黄色提示框
2. 显示暂停原因
3. 显示已完成的统计数据
4. 显示上次爬取位置

## 注意事项

1. **暂停延迟**
   - 点击暂停后需要等待当前学校爬取完成
   - 通常需要等待 10-30 秒
   - 系统会显示"暂停指令已发送"

2. **超时时间**
   - 默认为 15 分钟
   - 可根据需求修改
   - 建议不要太短，避免频繁暂停

3. **恢复机制**
   - 恢复后计时器重置
   - 从暂停位置继续
   - 不会重复爬取

4. **并发控制**
   - 同一时间只能有一个爬取任务
   - 暂停时不能开始新的爬取
   - 必须先恢复或停止当前任务
