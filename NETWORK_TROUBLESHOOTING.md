# 网络问题排查和改进说明

## 问题背景
生产环境中无法爬取学校数据，而本地环境可以正常工作。

## 已实施的改进

### 1. 增强网络请求容错性

#### 超时时间调整
- 将超时时间从 30 秒增加到 60 秒
- 生产环境网络可能更慢，需要更长的等待时间

#### 重试机制优化
- 保留最多 3 次重试
- 指数退避策略：3秒、6秒、9秒
- 增加更详细的错误诊断信息

#### 请求头部优化
- 添加完整的浏览器请求头部：
  - `Accept`: 接受的内容类型
  - `Accept-Language`: 语言偏好
  - `Connection`: keep-alive
  - `User-Agent`: 浏览器标识

### 2. 添加心跳机制
- 每 30 秒发送一次心跳事件
- 防止 SSE 连接被中间代理或负载均衡器超时关闭
- 提高长时间运行的稳定性

### 3. 网络连通性测试
- 在开始爬取前先测试网络连通性
- 如果测试失败，立即停止爬取并给出详细错误信息

### 4. 增强错误诊断
根据错误类型提供针对性的诊断信息：
- **DNS 解析失败**：提示域名无法访问或 DNS 配置问题
- **连接被拒绝**：提示目标服务器不可达或防火墙拦截
- **请求超时**：提示网络响应过慢或防火墙限制

### 5. 新增网络测试工具
- 添加独立的网络测试 API：`/api/test-network`
- 前端添加"测试网络"按钮
- 实时显示测试结果和诊断信息

## 如何使用网络测试工具

### 方法 1：通过前端界面
1. 访问学校管理页面
2. 点击"测试网络"按钮
3. 查看测试结果：
   - ✅ 成功：显示响应时间、状态码、内容长度
   - ❌ 失败：显示错误原因和诊断建议

### 方法 2：通过 API
```bash
curl -X POST http://your-domain/api/test-network \
  -H "Content-Type: application/json" \
  -d '{"url":"http://school.nseac.com/a/zkyx.html"}'
```

## 生产环境配置检查清单

如果测试失败，请检查以下配置：

### 1. 网络出口策略
- ✅ 确认允许出站 HTTP 请求（端口 80）
- ✅ 确认允许出站 HTTPS 请求（端口 443）
- ✅ 确认可以访问外部域名

### 2. 防火墙规则
- ✅ 检查安全组规则是否允许 HTTP/HTTPS 出站
- ✅ 检查是否有应用层防火墙拦截请求
- ✅ 检查是否有 IP 白名单限制

### 3. DNS 配置
- ✅ 确认 DNS 服务器配置正确
- ✅ 确认可以解析外部域名
- ✅ 测试 DNS 解析：`nslookup school.nseac.com`

### 4. 代理配置
- ✅ 如果使用代理，确认代理配置正确
- ✅ 确认代理允许访问目标域名
- ✅ 确认代理超时时间足够长（建议 > 60秒）

## 常见问题解决

### 问题 1：网络测试显示"连接被拒绝"
**原因**：防火墙或安全组阻止了出站请求
**解决**：
- 检查并修改防火墙规则，允许出站 HTTP 请求
- 如果使用云服务，检查安全组配置

### 问题 2：网络测试显示"请求超时"
**原因**：网络响应过慢或被网络设备拦截
**解决**：
- 检查网络带宽是否充足
- 检查是否有代理服务器导致延迟
- 增加超时时间（已调整为 60 秒）

### 问题 3：网络测试显示"DNS 解析失败"
**原因**：DNS 配置问题或域名被屏蔽
**解决**：
- 检查 DNS 服务器配置
- 尝试使用公共 DNS（如 8.8.8.8）
- 检查域名是否被网络策略屏蔽

## 技术细节

### API 端点
- **学校爬取**：`PATCH /api/schools` (SSE 流式返回)
- **网络测试**：`POST /api/test-network`

### SSE 事件类型
- `start`: 开始爬取
- `page`: 开始爬取指定页面
- `pageComplete`: 页面爬取完成
- `school`: 学校新增或更新
- `heartbeat`: 连接保持活跃（每 30 秒）
- `complete`: 爬取完成
- `error`: 发生错误

### 错误处理策略
1. 单个页面失败：记录错误，继续爬取下一页
2. 网络连通性测试失败：立即停止爬取
3. 连接中断：前端自动重连（刷新页面）

## 后续优化建议

1. **数据源多样化**：添加备用数据源，如果主源不可用自动切换
2. **缓存机制**：缓存已爬取数据，减少重复请求
3. **增量爬取**：只爬取更新的数据，而不是全量爬取
4. **分布式爬取**：使用消息队列实现分布式爬取，提高效率
5. **监控告警**：添加爬取成功率监控，失败时自动告警
